=== Output: docs/specs/ui.adoc (excerpt)

```asciidoc
= User Interface Requirements
:version: 1.0.0
:last-updated: 2026-02-20
:toc:

== UI-001
The system shall display products in a grid or list format.

Type:: Ubiquitous
Source:: US-001: View Product Catalog, Criterion 1
Verification:: Navigate to the product listing page. Confirm that products are displayed in a grid or list layout.

== UI-002
The system shall show the name, price, and image for each product in the listing.

Type:: Ubiquitous
Source:: US-001: View Product Catalog, Criterion 2
Verification:: View the product listing. Confirm each product displays its name, price, and image.

== UI-003
The system shall display an add-to-cart button on each product.

Type:: Ubiquitous
Source:: US-003: Add to Cart, Criterion 1
Verification:: View any product in the listing. Confirm an add-to-cart button is visible.
```

=== Output: docs/specs/error-handling.adoc

```asciidoc
= Error Handling Requirements
:version: 1.0.0
:last-updated: 2026-02-20
:toc:

== ERR-001
When the user submits a search query that matches no products, the system shall display a helpful empty-state message.

Type:: Event-driven
Source:: US-002: Search Products, Criterion 3
Verification:: Search for a term that matches no products. Confirm that a message is displayed indicating no results were found.

== ERR-002
If payment processing fails, then the system shall display an error message and allow the user to retry.

Type:: Unwanted behavior
Source:: US-005: Checkout Process, Criterion 6
Verification:: Attempt to complete checkout with payment details that cause a failure. Confirm that an error message is displayed and a retry option is available.
```

== Part 2: Sync Scenario

=== Change

User stories updated from version 1.0.0 to 1.1.0. Changes:
- *New* US-006: Order Tracking added under Checkout feature
- *Modified* US-002, Criterion 2: "display matching products" changed to "display matching products sorted by relevance"
- *Removed* US-001, Criterion 4: sort option criterion removed

=== Change Report

```
Spec sync: user-stories 1.0.0 -> 1.1.0

Added:
  - ORD-001: The system shall display order status for each placed order.
  - ORD-002: When the order status changes, the system shall display the updated status.
  - NOTIF-002: When the order status changes, the system shall send a notification to the user.

Modified:
  - SRCH-003: Added "sorted by relevance" to search results display

Removed:
  - SRCH-002: Source criterion (US-001, Criterion 4: sort option) removed

Categories:
  - Propose adding "Order Management" (ORD) -- 2 new requirements
  - No empty categories to remove

No changes to: UI, CART, CHK, ERR
```

=== Result After Sync

- `index.adoc`: version bumped to 2.0.0 (requirements removed), `:source-stories-version:` updated to 1.1.0
- `search.adoc`: version bumped to 2.0.0 (SRCH-002 removed, SRCH-003 modified)
- `order-management.adoc`: created at version 1.0.0 with ORD-001 and ORD-002
- `notifications.adoc`: version bumped to 1.1.0 (NOTIF-002 added)
- Traceability matrix updated to reflect all changes
