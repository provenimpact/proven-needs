= Example: Intent to Gherkin Feature Files

== Context

The orchestrator received the intent: "I want an e-commerce site where users can browse products, add them to cart, and checkout."

After feature decomposition, three feature packages were created:

* `product-browsing` (prefix: PROD)
* `shopping-cart` (prefix: CART)
* `checkout` (prefix: CHK)

This example shows the `needs-features` output for the `product-browsing` feature package, which produces two `.feature` files, plus the shopping-cart and checkout features.

== File 1: docs/features/product-browsing/product-catalog.feature

[source,gherkin]
----
@product-browsing
Feature: Product Catalog
  As a shopper,
  I want to browse available products,
  so that I can find items I want to purchase.

  Background:
    Given the following products exist:
      | name     | price | category    | image         |
      | Widget A | 9.99  | electronics | widget-a.jpg  |
      | Widget B | 19.99 | electronics | widget-b.jpg  |
      | Shirt    | 29.99 | clothing    | shirt.jpg     |

  @PROD-001
  Scenario: Products are displayed with details
    When I visit the product catalog
    Then I should see 3 products
    And each product should show its name, price, and image

  @PROD-001
  Scenario: Empty catalog shows no products
    Given no products exist
    When I visit the product catalog
    Then I should see 0 products

  @PROD-002
  Scenario: Each product shows name, price, and image
    When I visit the product catalog
    Then each product should show its name
    And each product should show its price
    And each product should show its image

  @PROD-003
  Scenario: Filter products by category
    When I visit the product catalog
    And I filter by category "electronics"
    Then I should see 2 products
    And I should see "Widget A"
    And I should see "Widget B"
    But I should not see "Shirt"

  @PROD-003
  Scenario: Filter by category with no matches
    When I visit the product catalog
    And I filter by category "books"
    Then I should see 0 products

  @PROD-004
  Scenario Outline: Sort products by attribute
    When I visit the product catalog
    And I sort by "<attribute>" "<direction>"
    Then the products should be ordered by <attribute> <direction>

    Examples:
      | attribute | direction  |
      | price     | ascending  |
      | price     | descending |
      | name      | ascending  |
----

== File 2: docs/features/product-browsing/product-search.feature

[source,gherkin]
----
@product-browsing
Feature: Product Search
  As a shopper,
  I want to search for products by keyword,
  so that I can quickly find specific items.

  Background:
    Given the following products exist:
      | name            | description                |
      | Blue Widget     | A high-quality blue widget |
      | Red Widget      | A standard red widget      |
      | Leather Jacket  | Premium leather jacket     |

  @PROD-005
  Scenario: Search input is available
    When I visit the product catalog
    Then I should see a search input

  @PROD-006
  Scenario: Search returns matching products
    When I search for "widget"
    Then I should see 2 products
    And I should see "Blue Widget"
    And I should see "Red Widget"
    But I should not see "Leather Jacket"

  @PROD-006
  Scenario: Search is case-insensitive
    When I search for "WIDGET"
    Then I should see 2 products

  @PROD-007
  Scenario: Search with no results shows empty state
    When I search for "nonexistent-product-xyz"
    Then I should see a message "No products found"

  @PROD-008
  Scenario: Search highlights matching terms
    When I search for "widget"
    Then the term "widget" should be highlighted in the results
----

== Constraint Overlap Example

A project-wide constraint exists in `docs/constraints.adoc`:

....
== Security
* All user input must be validated before processing.
....

While writing search scenarios, the behavior "search input is sanitized" would normally become a scenario. Since it duplicates the project-wide constraint, it is NOT included in the `.feature` file. Instead, a Gherkin comment notes the constraint:

[source,gherkin]
----
  # NOTE: Input validation for search queries is enforced by project
  # constraint (Security: "All user input must be validated before
  # processing"). Not duplicated here.
----

== Shopping Cart: docs/features/shopping-cart/cart-management.feature

[source,gherkin]
----
@shopping-cart
Feature: Cart Management
  As a shopper,
  I want to add products to my cart and manage quantities,
  so that I can purchase multiple items at once.

  Background:
    Given the following products exist:
      | name     | price |
      | Widget A | 9.99  |
      | Widget B | 19.99 |

  @CART-001
  Scenario: Add product to cart
    When I add "Widget A" to the cart
    Then the cart count should be 1
    And I should see a confirmation "Widget A added to cart"

  @CART-002
  Scenario: Adding duplicate product increments quantity
    Given "Widget A" is in my cart with quantity 1
    When I add "Widget A" to the cart
    Then "Widget A" should have quantity 2 in my cart

  @CART-003
  Scenario: View cart contents
    Given "Widget A" is in my cart with quantity 2
    And "Widget B" is in my cart with quantity 1
    When I view my cart
    Then I should see 2 distinct items
    And the cart total should be 39.97

  @CART-004
  Scenario: Update item quantity
    Given "Widget A" is in my cart with quantity 1
    When I change the quantity of "Widget A" to 3
    Then "Widget A" should have quantity 3 in my cart
    And the cart total should be 29.97

  @CART-005
  Scenario: Remove item from cart
    Given "Widget A" is in my cart with quantity 1
    And "Widget B" is in my cart with quantity 1
    When I remove "Widget A" from the cart
    Then I should not see "Widget A" in my cart
    And the cart total should be 19.99
----

== Checkout: docs/features/checkout/checkout-process.feature

[source,gherkin]
----
@checkout
Feature: Checkout Process
  As a shopper,
  I want to complete my purchase,
  so that I can receive my ordered items.

  Background:
    Given I have items in my cart totaling 49.98

  @CHK-001
  Scenario: Enter shipping address
    When I proceed to checkout
    Then I should see a shipping address form

  @CHK-002
  Scenario: Select payment method
    When I proceed to checkout
    And I enter a valid shipping address
    Then I should be able to select a payment method

  @CHK-003
  Scenario: Review order summary
    When I proceed to checkout
    And I enter a valid shipping address
    And I select a payment method
    Then I should see an order summary showing my items and total

  @CHK-004
  Scenario: Confirm order
    When I complete the checkout process
    Then I should see an order confirmation page
    And I should receive an order confirmation email

  @CHK-005
  Scenario: Payment failure
    When I complete the checkout process
    And the payment gateway returns an error
    Then I should see a payment error message
    And I should be able to retry the payment
----
