== Technical Context

Greenfield e-commerce web application. Users browse products, manage a shopping cart, and complete purchases. No existing codebase.

== Decisions and Constraints

* <<../adrs/0001-use-typescript.adoc#,ADR-0001>>: Use TypeScript for backend and frontend
* <<../adrs/0002-use-postgresql.adoc#,ADR-0002>>: Use PostgreSQL for persistence
* <<../adrs/0003-use-nextjs.adoc#,ADR-0003>>: Use Next.js as the fullstack framework
* <<../adrs/0004-use-stripe.adoc#,ADR-0004>>: Use Stripe for payment processing

ADR-0003 and ADR-0004 were created during Phase 0 of this design process.

== Research and Unknowns

=== Resolved

Framework selection:: Next.js chosen for server-side rendering (SEO for product pages) and API routes (backend endpoints in the same project). See ADR-0003.

Payment processing:: Stripe chosen for PCI compliance and hosted checkout. See ADR-0004.

=== Open Questions

None.

== System Design

=== Frontend

==== Product Catalog Page
Server-rendered page using Next.js. Fetches products from the API with pagination and search. Renders product cards with image, name, price, and "Add to Cart" button.

==== Cart Page
Client-side interactive page. Reads cart state from CartContext. Renders line items with quantity controls and a running total. "Proceed to Checkout" button navigates to the checkout flow.

==== Checkout Page
Multi-step form: shipping address, payment (Stripe Elements), order review. Submits order to the API on confirmation.

==== Shared Components
* `ProductCard` -- displays a single product summary
* `CartIcon` -- header icon showing item count from CartContext
* `SearchBar` -- text input with debounced search, triggers product catalog filtering

=== Backend (API Routes)

==== GET /api/products
Returns paginated product list. Supports `?q=` for text search and `?page=` for pagination.

==== GET /api/products/:id
Returns a single product with full details.

==== POST /api/cart/items
Adds an item to the cart. Body: `{ productId, quantity }`. Validates stock availability.

==== PATCH /api/cart/items/:id
Updates quantity of a cart item. Body: `{ quantity }`. Validates stock.

==== DELETE /api/cart/items/:id
Removes an item from the cart.

==== POST /api/checkout
Creates an order from the current cart. Validates stock, calculates totals, creates Stripe PaymentIntent, persists order. Returns order confirmation.

=== State Management

CartContext (React Context) holds the current cart state client-side. Synced with the backend on mutations. Persisted in localStorage for anonymous users.

=== External Integrations

Stripe:: Payment processing via Stripe SDK. Server-side: create PaymentIntents. Client-side: Stripe Elements for card input.

== Story Resolution

=== Story 1: View Product Catalog

Components:: Product Catalog Page, ProductCard, GET /api/products
Criteria::
* Display products in grid or list -> Product Catalog Page renders ProductCard components from GET /api/products (UI-001)
* Show product name, price, and image -> ProductCard component displays these fields (UI-002)
* Filter by category -> GET /api/products supports `?category=` parameter, Product Catalog Page renders category filter (SRCH-001)
* Sort by selected attribute -> GET /api/products supports `?sort=` parameter, Product Catalog Page renders sort control (SRCH-002)

=== Story 2: Search Products

Components:: SearchBar, Product Catalog Page, GET /api/products
Criteria::
* Search by keyword -> SearchBar sends `?q=` to GET /api/products, backend does text search on product name (SRCH-003)
* Display matching results -> Product Catalog Page re-renders with filtered results (SRCH-003)
* Show "no results" message -> Frontend displays empty state when API returns zero products (ERR-001)
* Highlight matching terms -> Frontend highlights search terms in ProductCard text (SRCH-003)

=== Story 3: Add to Cart

Components:: ProductCard, CartContext, CartIcon, POST /api/cart/items
Criteria::
* Display add-to-cart button -> ProductCard shows button on each product (UI-003)
* Update cart count -> POST /api/cart/items, CartContext updates, CartIcon reflects new count (UI-004, CART-001)
* Show success confirmation -> Frontend displays success message after add (UI-005)
* Increment quantity for duplicate -> POST /api/cart/items increments quantity if product already in cart (CART-002)

=== Story 4: View Cart

Components:: Cart Page, CartContext, PATCH/DELETE /api/cart/items
Criteria::
* Display cart items with quantities -> Cart Page reads CartContext and renders line items (UI-006)
* Display prices and total -> Cart Page computes and shows item prices and cart total (UI-007)
* Update total on quantity change -> PATCH /api/cart/items/:id, CartContext updates, Cart Page re-renders total (CART-003)
* Remove item and update total -> DELETE /api/cart/items/:id, CartContext updates, Cart Page removes item and re-renders total (CART-004)

=== Story 5: Checkout Process

Components:: Checkout Page, POST /api/checkout, Stripe integration, email service
Criteria::
* Shipping address form -> Checkout Page step 1: address form with validation (CHK-002)
* Payment method selection -> Checkout Page step 2: Stripe Elements card input (CHK-003)
* Order summary before confirmation -> Checkout Page step 3: order summary with line items and totals (CHK-004)
* Display confirmation page -> POST /api/checkout returns order ID, frontend displays success page (CHK-001)
* Send confirmation email -> POST /api/checkout triggers email via transactional email service after successful payment (NOTIF-001)
* Handle payment failure -> If Stripe PaymentIntent fails, display error and allow retry (ERR-002)

= Data Model
:version: 1.0.0
:last-updated: 2026-02-20
:toc:

== Product

[cols="1,1,2", options="header"]
|===
| Field | Type | Notes

| id | UUID | Primary key
| name | VARCHAR(255) | Required, indexed for search
| description | TEXT | Optional
| price | DECIMAL(10,2) | Required, >= 0
| image_url | VARCHAR(500) | Required
| stock_quantity | INTEGER | Required, >= 0
| created_at | TIMESTAMP | Auto-set
| updated_at | TIMESTAMP | Auto-updated
|===

== Cart

[cols="1,1,2", options="header"]
|===
| Field | Type | Notes

| id | UUID | Primary key
| session_id | VARCHAR(255) | For anonymous users, indexed
| created_at | TIMESTAMP | Auto-set
| updated_at | TIMESTAMP | Auto-updated
|===

== CartItem

[cols="1,1,2", options="header"]
|===
| Field | Type | Notes

| id | UUID | Primary key
| cart_id | UUID | FK -> Cart, indexed
| product_id | UUID | FK -> Product
| quantity | INTEGER | Required, >= 1
|===

== Order

[cols="1,1,2", options="header"]
|===
| Field | Type | Notes

| id | UUID | Primary key
| cart_id | UUID | FK -> Cart
| status | ENUM | pending, paid, shipped, cancelled
| total | DECIMAL(10,2) | Computed at checkout
| shipping_address | JSONB | Address fields
| stripe_payment_id | VARCHAR(255) | Stripe PaymentIntent ID
| created_at | TIMESTAMP | Auto-set
|===

== OrderItem

[cols="1,1,2", options="header"]
|===
| Field | Type | Notes

| id | UUID | Primary key
| order_id | UUID | FK -> Order, indexed
| product_id | UUID | FK -> Product
| quantity | INTEGER | Snapshot from CartItem
| unit_price | DECIMAL(10,2) | Snapshot from Product at time of order
|===

== Relationships

* Cart 1--* CartItem
* CartItem *--1 Product
* Order 1--* OrderItem
* OrderItem *--1 Product
* Order 1--1 Cart (order created from cart)
